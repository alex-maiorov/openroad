# OpenROAD Database Log Analysis Tooling

This directory contains Python libraries for analyzing SQLite database logs generated by OpenROAD tools. It is designed to be **Pythonic**, **minimal**, and **Pandas-native**, avoiding convoluted dictionary structures while remaining fully self-documenting.

## Architecture

* `core.py`: Contains the `DatabaseLog` wrapper for SQLite and the `AnalysisModule` base class. Tables are queried and returned as **Pandas DataFrames**.
* `<tool>.py` (e.g. `exa.py`, `gpl.py`): Tool-specific modules that provide unified APIs for raw and derived metrics.
* **Tuple Returns**: Properties and methods returning datasets always return a `(DataFrame, dict)` tuple, where the dictionary maps column names to their descriptions.

## Usage Example

```python
from tools.database_log_analysis import DatabaseLog, ExaAnalysis

# 1. Connect to the database
db = DatabaseLog("tmp/exa_test.db")

# 2. Instantiate the tool-specific analysis module
exa = ExaAnalysis(db)

# 3. Print out self-documenting descriptions of available metrics
ExaAnalysis.describe()

# 4. Fetch raw data and its column descriptions
df, desc = exa.st_bulk
print(f"Data types:\n{df.dtypes}")
print(f"Description for 'val': {desc['val']}")

# 5. Fetch contrived/derived metrics
magic_df, magic_desc = exa.calculate_magic_metric()
print(magic_df.head())
```

## Adding New Tools

To add analysis for a new OpenROAD tool:
1. Create `your_tool.py`.
2. Inherit from `AnalysisModule`.
3. Use `@property` for raw database tables, and regular methods for derived/calculated metrics.
4. Always return `(df, desc)` tuples. Use well-written docstrings, as they are automatically parsed by the `.describe()` method to help users.
